# Ikigai + Skillshare + LinkedIn Skill Assessments

### A jornada do Dev Padawan na Stack Mainframe ☕🚀

Imagine um **Padawan da tecnologia** iniciando sua jornada.

Ele abriu o primeiro **ISPF**, escreveu seu primeiro **COBOL**, submeteu seu primeiro **JCL** e aguardou ansiosamente o resultado do JOB.

Algumas vezes aparece:

```
RC=0000
```

Outras vezes aparece:

```
S0C7
```

Faz parte da jornada.

Mas além de aprender tecnologia, um profissional moderno precisa entender **como evoluir na carreira**.

Aqui entram três frameworks muito interessantes:

1️⃣ **Ikigai Canvas**
 2️⃣ **Skillshare**
 3️⃣ **LinkedIn Skill Assessments**

Eles representam **três dimensões diferentes da evolução profissional**.

------

# 1️⃣ Ikigai — Descobrir seu propósito

4

O **Ikigai** responde à pergunta fundamental:

> **Por que você faz o que faz?**

Ele conecta quatro elementos:

- O que você **ama fazer**
- No que você **é bom**
- O que o **mundo precisa**
- Pelo que você pode **ser pago**

Para um **Dev Padawan**, isso pode significar descobrir:

- paixão por **arquitetura de sistemas**
- interesse por **segurança**
- gosto por **programação**
- fascínio por **infraestrutura crítica**

No universo mainframe isso pode se traduzir em especializações como:

- desenvolvimento COBOL
- engenharia de performance
- arquitetura de integração
- segurança RACF
- automação de batch

O Ikigai é **a bússola da carreira**.

Ele responde:

> “Qual caminho devo seguir?”

------

# 2️⃣ Skillshare — Aprender e compartilhar conhecimento

4

Se o Ikigai define **propósito**, o Skillshare define **como o conhecimento circula**.

No mundo da tecnologia, aprender sozinho é praticamente impossível.

A evolução acontece quando:

- alguém aprende
- alguém ensina
- alguém compartilha

No universo mainframe isso sempre existiu.

Antigamente acontecia assim:

👨‍🏫 especialista sênior
 ➡ ensinava
 🧑‍💻 programador júnior

Hoje acontece através de:

- cursos online
- newsletters
- comunidades técnicas
- vídeos
- labs
- mentorias

É exatamente esse espírito que forma comunidades como:

- grupos de estudo
- fóruns técnicos
- conteúdos educacionais

Skillshare responde à pergunta:

> **Como o conhecimento se espalha?**

------

# 3️⃣ LinkedIn Skill Assessments — Validar habilidades

4

Aqui entramos na dimensão **profissional e de mercado**.

O **LinkedIn Skill Assessment** responde a outra pergunta:

> **Como provar que você sabe fazer algo?**

Muitos profissionais dizem:

- “Eu sei programar”
- “Eu sei banco de dados”
- “Eu sei cloud”

Mas o mercado precisa de **evidências**.

Os Skill Assessments oferecem:

- testes rápidos
- validação técnica
- badges no perfil

Eles funcionam como um **sinal de competência** para recrutadores.

------

# A relação entre os três frameworks

Agora vem a parte interessante.

Esses três conceitos formam **um ciclo completo de desenvolvimento profissional**.

```
IKIGAI
(descobrir propósito)

     ↓

APRENDIZADO / SKILLSHARE
(adquirir conhecimento)

     ↓

VALIDAÇÃO
(LinkedIn Skill Assessments)

     ↓

OPORTUNIDADES DE CARREIRA
```

Ou seja:

**Ikigai define a direção**
 **Skillshare acelera o aprendizado**
 **Skill Assessments validam o conhecimento**

------

# O impacto na vida do Dev Padawan

Para um desenvolvedor iniciante, esse ciclo muda completamente a trajetória profissional.

### 1️⃣ Clareza de carreira

O Ikigai ajuda o dev a entender:

- qual área gosta
- onde quer se especializar
- qual tipo de problema quer resolver

------

### 2️⃣ Aprendizado acelerado

Skillshare cria um ambiente onde:

- aprender é mais rápido
- erros são compartilhados
- boas práticas se espalham

------

### 3️⃣ Reconhecimento profissional

Skill Assessments ajudam a:

- construir reputação
- provar habilidades
- atrair oportunidades

------

# O verdadeiro segredo da jornada

O mais interessante é que esses três frameworks **não são etapas isoladas**.

Eles formam um ciclo contínuo.

Um profissional evolui assim:

1️⃣ Descobre novos interesses
 2️⃣ Aprende novas habilidades
 3️⃣ Valida conhecimentos
 4️⃣ Compartilha com outros

E então o ciclo começa novamente.
